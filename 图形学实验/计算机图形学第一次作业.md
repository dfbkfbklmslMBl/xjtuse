void DDAline(int x0, int y0, int x1, int y1, int color)
{
    int x;
    float dx, dy, y, k;
    dx = x1 - x0;
    dy = y1 - y0;
    k = dy/dx;
    y = y0;
    for (x = x0; x <= x1; x++)
    {
        drawpixel (x, int(y + 0.5), color);
        y = y + k;
    }
}

中点画线

```cpp
void Midpoint Line(int x0, int y0, int x1, int y1, int color)
{
    int a, b, d1, d2, d, x, y;
    a = y0 - y1;
    b = x1 - x0;
    d = 2 * a + b;
    d1 = 2 * a;
    d2 = 2 * (a + b);
    x = x0;
    y = y0;
    drawpixel(x, y, color);
    while (x < x1)
    {
        if (d < 0)
        {
            x++, y++, d += d2;
        }
        else
        {
            x++, d += d1;
        }
        drawpixel (x, y, color);
    }
}
```

**Bresenham 算法**

```cpp
void Bresenhamline (int x0,int y0,int x1, int y1,int color)
{   int x, y, dx, dy;
    float k, e;
    dx = x1-x0, dy = y1- y0, k=dy/dx; 
    e=-0.5, x=x0, y=y0;
    for (i=0; i<=dx; i++)
    {    drawpixel (x, y, color);
          x=x+1，e=e+k;
          if (e>=0)
          { y++, e=e-1;}
    }
  }
```

Bresenham算法主要基于整数运算，仅涉及整数加减法与简单比较判断，避免了浮点数运算开销，运行效率高，尤其适用于需大量绘制直线、对实时性要求高的图形场景。 数值微分法（DDA算法）利用直线斜率确定坐标增量，虽思路简单但存在浮点数运算，相对耗时，效率低于Bresenham算法，适合简单且对效率要求不极致的少量直线绘制情况。 直接用直线斜截式方程的简单实现方式，大量使用浮点数运算，计算速度慢且易有精度问题，效率最低，常用于对效率和精度要求均不高的简单临时绘图任务。 

分析结论：总体而言，Bresenham算法运行效率最优，数值微分法次之，直接用直线斜截式方程的方式效率最差
